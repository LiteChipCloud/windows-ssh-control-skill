# 03 安全边界与审计设计

## 1. 安全目标

1. 默认不依赖共享高权限账号。
2. 避免凭据、私钥、敏感路径进入仓库。
3. 对关键远程操作保留审计证据。

## 2. 最小权限建议

| 场景 | 建议账户 |
|---|---|
| 日常巡检 | 普通 SSH 账户 |
| 配置修复 | 临时提权账户 |
| 系统级变更 | 管理员账户（审批后） |

## 3. 凭据管理边界

1. 使用独立 SSH key，不复用个人主密钥。
2. 不在脚本中硬编码密码或 token。
3. 公钥授权脚本只用于初始化，不作为日常执行入口。

## 4. 审计建议

1. 记录命令摘要、执行时间、执行人。
2. 文件传输建议记录源路径、目标路径、文件摘要。
3. 对批量操作增加变更审批与回滚计划。

## 5. 开源发布前检查

1. 执行 CI 中的敏感信息扫描。
2. 人工检查 README/脚本示例是否仍为占位值。
3. 确认无私有域名、真实账号、真实路径泄露。
